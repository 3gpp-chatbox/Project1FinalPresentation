Project Overview

The project aimed to develop a Minimum Viable Product (MVP) that leverages Large Language Models (LLM) to extract and visualize state machines from 3GPP technical specifications.

Procedure Selection

Interactive browsing and selection of network procedures from 3GPP specifications

Flow Diagram Generation

Automatic conversion of JSON procedure data into Mermaid.js flow diagrams

JSON Data Viewer

Raw procedure data visualization for detailed technical analysis

Dynamic Visualization

Interactive and responsive UI for enhanced user experience

Key Features

Procedural Model Extraction

  • States: EMM-REGISTERED
  • Actions: Attach Request, Identity Response
  • Events: UE initiates Attach
  • Parameters: IMSI, GUTI
  • Conditionals: "If GUTI is unknown..."
  • Metadata: Procedure name, section number

Flow Property Graph Construction

  • Nodes: States and Events
  • Edges: Actions and Transitions
  • Properties: Parameters, Conditionals, Metadata

Visualization & Interaction

  • Graph Rendering: Display the extracted knowledge graph
  • Mermaid Integration: Generate a Mermaid code based diagram for a structured procedural view

Methodology and Approach

Research Methodology

  • Focused on extracting NAS (Non-Access Stratum) procedures for 5G NR from 3GPP TS 24.501
  • Selected Initial Registration and Periodic Registration update as key procedures for analysis
  • Used the latest version of the document (TS 24.501 v19.1.1, December 2024)

System Architecture & Implementation

Backend (FastAPI - Python)

Developed APIs to serve procedure data from structured JSON

Frontend (React.js)

  • Procedure Selection List (NAS procedures: Initial & Periodic Registration)
  • JSON Viewer to inspect raw procedure data
  • Mermaid.js Diagrams to visualize NAS signaling flows dynamically

API Endpoints

  • Listing available result sets and procedures
  • Fetching detailed NAS procedure graphs
